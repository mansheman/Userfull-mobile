--------------------------------------------------------------------------------

[PATH_TO_APK]

- Hardcoded secrets

PROMPT 2: Analisis Scan yang Sudah Ada
--------------------------------------------------------------------------------

Query database MobSF untuk scan dengan MD5 hash:
[MD5_HASH]

Berikan analisis lengkap meliputi:
1. App summary (nama, package, versi, ukuran)
2. Semua temuan HIGH dari manifest analysis
3. Daftar permission dangerous beserta deskripsinya

5. Hardcoded secrets yang ditemukan

8. Network security issues
9. Skor keamanan keseluruhan
10. Rekomendasi prioritas perbaikan

--------------------------------------------------------------------------------
PROMPT 4: Security Assessment Report (Format Lengkap)
--------------------------------------------------------------------------------

Buat laporan security assessment profesional untuk scan MobSF dengan hash:
[MD5_HASH]

Format laporan:
===========================================================
MOBILE APPLICATION SECURITY ASSESSMENT
===========================================================

1. EXECUTIVE SUMMARY
   - App identification
   - Risk level
   - Key findings in 1 paragraph

2. APPLICATION DETAILS
   - Package name, version, size
   - Hashes (MD5, SHA1, SHA256)
   - Min/Target SDK
   - Main activity

3. MANIFEST ANALYSIS
   - Table: Severity | Rule | Title | Description

4. PERMISSIONS ANALYSIS
   - Table: Permission | Level | Risk | Description

5. EXPORTED COMPONENTS
   - List of activities, services, receivers, providers
   - Risk assessment per component

6. CODE ANALYSIS
   - Hardcoded secrets
   - Certificate analysis
   - APKiD results

7. PRIVACY & TRACKING
   - Trackers detected
   - Data collection concerns

8. NETWORK SECURITY
   - Configuration issues
   - Domain/URL analysis

9. SBOM (Software Bill of Materials)
   - Libraries and versions

10. RECOMMENDATIONS
    - Prioritized list of fixes
    - High: immediate action
    - Warning: short-term fix
    - Info: best practice improvement

11. COMPLIANCE NOTES
    - OWASP MASVS mapping
    - GDPR/privacy considerations

--------------------------------------------------------------------------------
PROMPT 6: Privacy Audit
--------------------------------------------------------------------------------

Analisis aspek privacy dari hasil scan MobSF [MD5_HASH]:
1. Tracker yang terdeteksi (nama, kategori, URL)
2. Permission yang berhubungan dengan data privacy:
   - Location (ACCESS_*_LOCATION)
   - Contacts (READ_CONTACTS, GET_ACCOUNTS)
   - Storage (WRITE/READ_EXTERNAL_STORAGE)
   - Camera/Mic (CAMERA, RECORD_AUDIO)
   - SMS/Phone (SEND_SMS, READ_SMS, READ_PHONE_STATE)
3. Data yang dikirim ke pihak ketiga (domain/URL dari code analysis)
4. Apakah ada hardcoded PII atau credentials
5. Rekomendasi privacy compliance

PROMPT 8: Malware Indicator Check
--------------------------------------------------------------------------------

Periksa indikator malware dari scan MobSF [MD5_HASH]:
1. APKiD analysis (anti-vm, obfuscation, packer detection)
2. Permission yang sering disalahgunakan malware
3. Suspicious activities/services
4. Domain/IP mencurigakan
5. Base64 encoded strings
6. Native library analysis
7. Kesimpulan: apakah ada indikasi malicious

--------------------------------------------------------------------------------
PROMPT 9: OWASP MASVS Compliance Check
--------------------------------------------------------------------------------

Evaluasi hasil scan MobSF [MD5_HASH] terhadap OWASP MASVS (Mobile Application Security Verification Standard):

MASVS-STORAGE: Data storage security
MASVS-CRYPTO: Cryptography
MASVS-AUTH: Authentication & Session Management
MASVS-NETWORK: Network Communication
MASVS-PLATFORM: Platform Interaction
MASVS-CODE: Code Quality & Build Settings
MASVS-RESILIENCE: Resiliency Against Reverse Engineering

Berikan status PASS/FAIL per kategori dengan bukti dari scan result.
